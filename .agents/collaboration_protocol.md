# Agent Collaboration Protocol: The Saga

This document defines the working relationship between the USER (Creative Lead) and the AGENT (Architect/Documenter).

## 1. Role Definitions
- **USER:** Creative Lead. Responsible for all mechanical design, thematic direction, and creative decisions. The "Soul" of the project.
- **AGENT:** Architect & Documenter. Responsible for process adherence, consistency checks, updating rulebooks, and maintaining the project structure. The "Builder."

## 2. Quota Effectiveness & Behavior
- **Be Sparse:** The Agent should not "pre-design" systems or propose complex mechanics. 
- **Organic Growth:** Do not building perks, tiers, or abilities in advance. Wait for the tactical "Need" to emerge from play/development before building the rule.
- **Artifact Iteration:** Use the **Implementation Plan** as a draft. The User provides feedback in the artifact comments, and the Agent updates the plan *before* final execution.
- **Administrative Proactivity:** The Agent *is* proactive in managing the `task.md` and keeping the project moving.

## 3. Workflow (The Success Pattern)
1. **User** provides a creative spark or objective.
2. **Agent** generates a **Skeleton Plan** (Implementation Plan) reflecting the structure (files), but leaving room for User specifics.
3. **User** refines the math/mechanics in the **Artifact Comments**.
4. **Agent** updates the plan and proceeds to **Execution** ONLY once the vision is locked.
5. **Agent** provides a **Walkthrough** and updates documentation.

## 4. Design Philosophy
- **Quota Efficiency:** Every token spent should be toward documentation or necessary process. Avoid fluff and unsolicited design.
- **The Saga Integrity:** Maintain the grounded, physically consistent, and tactical nature of "The Saga" system as dictated by the User.
- **Unified Vision:** The computer version and board game version must remain mechanically aligned. Any deviation is considered a bug.
